# Personal Context Agent 需求文档

> 本文档描述 **要做什么**(What / Why)。技术实现见《实施规划文档》。
> 本版基于与作者的多轮设计讨论重写,取代旧版《Personal Context Agent 设计文档》。

---

## 1. 一句话定义

一个**飞书优先的对话型个人 Agent**,基于 pi-agent-core 运行时。它不是日记 App,也不是任务执行平台,而是**一个很懂我、能跟我交流的厉害朋友**:我把日记、想法、有意思的知识丢给它,它读懂我、回应我,并在后台沉淀出越来越准的"关于我"的长期理解。

> 日记只是输入源。**记忆和长期上下文才是产品本体。**

---

## 2. 目标用户

单用户,即作者本人(下称"我")。个人自用,本地优先,不做多租户、不做 SaaS。所有交互通过飞书(私聊 / 群)发生。

---

## 3. 产品角色

它应该是:

- 私人思想读者 / 长期自我上下文 Agent / 懂我的厉害朋友 / 第二大脑旁白员。

它要能做到:

- 接住情绪,但不把每次吐槽都做成总结报告。
- 从一坨想法里拆出主线、暗线和真正卡住的问题。
- 记得我长期在做什么、反复关心什么、讨厌什么。
- 在我推项目时,帮忙压实目标、验收标准和下一步判断。
- 在我逻辑自洽但行动卡住时,直接指出盲点。

它**不应该**输出:泛泛鼓励、鸡汤、固定模板、过度心理分析、没有证据的"我觉得你是某种人"。

---

## 4. 核心设计哲学:记忆

这是整个产品的灵魂。我们采用 **Hermes Agent 为主的混合记忆模型**,而不是传统的"数据库 + 每次检索注入"模型。

**关键洞察**(来自 Hermes):记忆不是事实的档案库,而是**一个关于"我"的、压缩过的、始终在场的工作模型**。它应该小到能一直待在上下文里、精选到不产生噪音、有纪律到知道该忘什么。

但 Hermes 原生的 ~3600 字上限对重度写作者(我每周可能有数千字语音转写日记)太小。所以我们**保留 Hermes 的纪律(有界、强制合并、会主动忘),但把预算抬高一个量级,并分层**。

### 4.1 四层记忆

| 层 | 内容 | 变化速度 | 是否常驻上下文 |
|---|---|---|---|
| **① 身份画像(prose)** | 我是谁、对什么好奇、价值观、判断习惯、表达风格、稳定的关系 | 慢(周更,保守) | **始终注入** |
| **② 工作集(结构化)** | 在做的项目、未闭环问题,各自成条(主旨 / 当前问题 / 已定的事 / 下一步 / 相关人),带 active/dormant 状态 | 中 | **仅 active 的注入** |
| **③ 最近 episode** | 最近 N 篇日记蒸馏出的结构化摘要 | 快 | 注入(给连续性) |
| **④ 原文 + episode 归档** | 全部日记原文与 episode | — | 不注入,按需全文检索(FTS) |

**身份画像 vs 工作集的分工**(作者的核心要求):"我这个人不会因为短期做了个新东西就变了。" 身份画像是稳定的"我",绝不因本周冒出个新项目就改写;工作集是"我最近在搞什么",项目黄了就删掉那条,画像纹丝不动。

### 4.2 写入机制:分层有别的写权限

不是"所有记忆都自由写",而是**按层区分写权限**——身份画像是慢变量,必须被保护;工作集是快变量,可以随手改。

| 层 | 谁能写 | 写的时机 | 纠错方式 |
|---|---|---|---|
| **① 身份画像** | 仅周度合并 / 显式纠错命令 | 周更保守;**绝不在单篇日记里被写** | `/profile` 查看,一句话纠正;每次改都留审计 |
| **② 工作集** | agent | 日记轮内可实时 upsert(低风险、结构化、可单条删) | `/working` 查看,一句话删/改 |
| **③ episode** | agent | 每篇日记落一条 | 可追溯到原文 |

- **画像受保护**(P1 安全边界):单篇日记**不能**改写身份画像,避免一时情绪/新项目污染长期的"我"。画像只在每周合并出现跨篇稳定信号时、或我显式纠错时才变,且每次变更都记一条审计(旧值/新值/来源 episode 与日记 id/原因)。
- **工作集自由写**:agent 在对话中可随手 upsert 工作集条目,不设审阅队列;每次写入在飞书卡片上留一行轻通知,如 `💾 已更新:叶佳老大项目 → 下一步`。
- 记忆很小、随时看得全(`/profile`、`/working`),看到不对**一句话就能纠正**("把焦虑那条删了")。
- **有界纪律**:工作集结构化、画像周更保守,都不会无限膨胀。
- **高风险红线**(靠 prompt 强约束,不是靠审阅):不写心理状态定性、不写关系/健康/人格结论、不写未经确认的敏感推断。只写基于文本的观察。
  - 坏:`你有焦虑倾向`
  - 好:`最近几篇日记里,你反复提到对项目落地不确定性的担心`

### 4.3 消化管线

- **每篇日记** → agent 顺手蒸馏出一条 **episode**(它本来就在读这篇,几乎零额外成本)。episode 字段沿用作者上一代实现(WeChatWS):`facts / emotions / thoughts / blind_spots / actions / long_term_memory_candidates / brief`,每条带证据与置信度。日记轮可顺带 upsert 工作集,但**不碰身份画像**。
- **每周日 23:55** → 定时合并:读本周所有 episode,**增量更新**工作集(不再像旧版全量重写画像而丢失细节),并**保守更新**身份画像(出现跨篇稳定信号才动,以旧画像为基)。本周没有日记就直接跳过。

### 4.4 可追溯

所有 episode、工作集条目、画像更新都能回溯到原始日记。原文永远完整保存,绝不丢失。**身份画像的每次变更额外记一条审计**(旧值/新值/触发来源的 episode 与日记 id/原因/合并批次),以便事后判断某次画像变化来自哪些证据。

## 5. 使用场景

标注 **[v1]** 为第一版必须;**[后续]** 为北极星方向,架构需预留但暂不实现。

1. **[v1] 日记群**:我把日记发到专门的日记群。agent 回复我(流式,像朋友聊天),同时在后台写一条 episode。日积月累它越来越懂我。
2. **[v1] 周度总结**:每周日 23:55 定时任务读本周所有日记,产出一份本周总结发到日记群,并顺带结构化整理、更新工作集与画像。
3. **[v1] 记日记提醒**:每天一次定时检查,如果我超过 3 天没记日记,发一条提醒到日记群。
4. **[后续] 主题群**:私聊里 `/new-chat 主题` 创建一个新群,围绕某一件事持续深聊。该群不设空闲超时,靠上下文 85% 自动 compact 维持长会话。
5. **[后续] 知识讨论**:我把有意思的知识转发到私聊跟它讨论。需要联网搜索、抓取文章正文(先简单做,后续专项优化)。
6. **[后续] 写飞书文档**:私聊中觉得某段讨论有价值时,用 `/write_doc <指令>` 让它整理成飞书云文档(仅此命令触发,agent 不自作主张)。
7. **[后续] 通用定时脚本 + 通知群**:可配置定时运行的脚本(如 AI 日报、知识卡片),执行结果转发到一个通知群。v1 需把"可配置定时任务"的骨架立起来,但只填周总结/提醒两条内置任务。

## 6. 交互形态

### 6.1 Chat 类型

| 类型 | 用途 | 创建方式 |
|---|---|---|
| 日记群 | 记日记、收周总结、收提醒 | `/new-diary-group` |
| 私聊 DM | 日常讨论、知识讨论、控制命令 | 默认存在 |
| 主题群 | 围绕一件事持续深聊 | `/new-chat 主题` |
| 通知群 | 接收定时任务转发结果(无对话) | 配置 |

### 6.2 回复风格(自动选择四种模式)

agent 不输出固定模板,根据当前内容自动选模式:

- **接住型**:我只是吐槽 / 释放情绪 → 短,不教育,不强行给方案。
- **拆线型**:一坨想法混在一起 → 拆主线、指暗线、抓一个最值得处理的问题。
- **反驳型**:逻辑有坑 / 明显自欺 → 直接、有证据、不羞辱。
- **沉淀型**:出现重要判断 / 产品洞察 → 写成可复用观点,沉淀进记忆。

### 6.3 斜杠命令(控制面,只在显式命令时触发)

| 命令 | 作用 |
|---|---|
| `/new-diary-group` | 创建日记群 |
| `/new-chat <主题>` | 创建主题群 |
| `/new` | 重置当前会话 |
| `/compact` | 压缩当前会话上下文 |
| `/write_doc <指令>` | 把当前讨论整理成飞书文档 |
| `/consolidate` | 手动触发周度合并 |
| `/profile` `/working` | 查看当前画像 / 工作集(只读,配合自然语言纠错) |

---

## 7. 非目标

- 不做通用任务执行平台:不发邮件、不跑系统命令、不自动浏览网页执行操作、不创建外部任务。工具仅限**理解/记忆、联网搜索、抓取文章、写飞书文档**。
- 不把一时情绪写成长期人格结论。
- 不做心理诊断、健康判断、高风险关系定性。
- 不做多用户 / SaaS / 复杂权限。
- 不把所有历史原文直接塞进 prompt(原文只做按需 FTS 检索)。

---

## 8. 关键风险与缓解

| 风险 | 缓解 |
|---|---|
| **记忆污染**(一时情绪变成长期人格) | 身份画像**永不在单篇日记里被写**,只由周合并/显式纠错改且全程审计;高风险红线靠 prompt 禁止;画像周更保守不被短期事件带偏;工作集结构化可单条删除;一切可追溯、可废弃 |
| **朋友感变模板感** | 四种回应模式自动选择;一次只抓一个重点;不强制完整分析 |
| **过度心理分析** | prompt 禁止诊断;高风险推断只留文本观察;我可标记"不要这样解读我" |
| **重度写作淹没上下文** | 四层 + 消化管线;原文不进 prompt;长会话 autocompact |
| **记忆确认成本高** | 不设审阅队列;自由写 + 轻通知 + 随时一句话纠正 |

---

## 9. 范围边界

- **v1 交付**:日记群完整链路(记日记 → 流式回复 → 写 episode → 周总结 → 提醒)+ 四层记忆 + 消化管线 + 基础斜杠命令。
- **北极星**:多功能个人助手(主题群深聊、知识讨论、写文档、通用定时脚本、通知群)。架构预留 chat 类型与定时任务表,但 v1 不提前写这些功能的具体逻辑。
- 历史日记数据由作者在系统建成后单独一次性导入,不在 v1 范围内自动处理。
